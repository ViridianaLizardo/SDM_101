Introducción a Maxent

MaxEnt, abreviación de Maxima Entropía, es un método popular para modelar nichos y distribuciones. Su popularidad se origina de:

  • Funciona bien con datos de sólo presencia.

  • Genera datos de idoneidad

  • Es familiar para múltiples personas.

Conceptos clave

  • Sólo presencia: No requiere datos de ausencia, genera “pseudo ausencias” a partir del espacio disponible en el área de entrenamiento.
  • Pseudo-ausencias: Puntos generados artificialmente donde no se ha registrado la especie. Proveen un contraste para mejorar la precisión del modelo.
  • Área de entrenamiento: Total de datos a partir de los cuales hará inferencias. Usualmente es equivalente a la M, pero puede no serlo. Tambien llamado background.
  • Regularización: Método para evitar el sobreajuste ajustando la complejidad del modelo. Se modifica con la tasa de regularización (se da por el regularization multiplier).
  • AUC (Area Under the Curve): Métrica por default que mide la capacidad del modelo para distinguir entre presencias y ausencias. Va de 0.5 (azar) a 1 (perfecta discriminación).
  • Idoneidad: Mapas que muestran la probabilidad o idoneidad de presencia de una especie en diferentes áreas geográficas. Resultado del Maxent.
  • Validación cruzada: Técnica de validación que divide los datos en subconjuntos para entrenar y evaluar el modelo varias veces. Estimar la precisión del modelo y su capacidad de generalización.

Features de Maxent

En Maxent, las “features” (características) son funciones matemáticas que transforman las variables ambientales de entrada en el modelo. El default es ponerlas todas, que se expresa como “lpqh”.

Linear (l)

Relaciona la variable ambiental de forma directa y proporcional.

Función: \(f(x) = x\)

Product Feature (p)

Modela interacciones multiplicativas entre variables.

Función: \(f(x, y) = x \cdot y\)

Quadratic Feature (q)

Incluye términos cuadráticos de la variable ambiental.

Función: \(f(x) = x^2\)

Hinge Feature (HF)

Permite cambios abruptos en la pendiente de la función.

Función: \(f(x) = max(0, x - \text{threshold})\)

Threshold Feature (t)

Actúa como un umbral binario.

Función: \(f(x) = I(x > \text{threshold})\)

Regularization multiplier

La regularización en MaxEnt previene el sobreajuste del modelo y penaliza modelos que son demasiado complicados. Usa el parámetro de regularización(regularization o beta multiplier):

  • Valores bajos de regularización (por ejemplo, 0.5) permiten que el modelo sea más complejo y se ajuste más estrechamente a los datos de entrenamiento, llevando a un sobreajuste..
  • Valores altos de regularización (por ejemplo, 2 o más) simplifican el modelo. El modelo puede no capturar relaciones importantes en los datos, llevando a un subajuste.
  • Default: 1

Evaluación de Modelos MaxEnt usando Validación Cruzada

Esta evaluación permite determinar qué tan bien el modelo predice la presencia de especies en nuevas ubicaciones y condiciones. Se usa la validación cruzada que estima el rendimiento del modelo comparando errores a partir de un set de datos no usado para crear el modelo (‘testing’).

Conceptos clave

Sensibilidad:Proporción de verdaderos positivos detectados por el modelo.

\(Sensibilidad = \frac{TP}{TP + FN}\).

Especificidad: Proporción de verdaderos negativos detectados por el modelo.

\(Especificidad = \frac{TN}{TN + FP}\).

Omisión: Error que ocurre cuando el modelo no predice una presencia conocida (falso negativo).

Comisión: Error que ocurre cuando el modelo predice una presencia en una ubicación donde la especie no está presente (falso positivo).

Tabla de Confusión

Matriz que resume el rendimiento de un modelo de clasificación mostrando los recuentos de verdaderos positivos (TP), verdaderos negativos (TN), falsos positivos (FP) y falsos negativos (FN).

Permite calcular métricas de evaluación como sensibilidad, especificidad, precisión, y TSS.

Indices

True Skill Statistic (TSS)

El True Skill Statistic (TSS) es una métrica que combina la sensibilidad (tasa de verdaderos positivos) y la especificidad (tasa de verdaderos negativos). Se calcula como:

\[ TSS = Sensibilidad + Especificidad - 1 \]

Valores: Un TSS de 1 indica una predicción perfecta, mientras que un TSS de 0 indica una predicción no mejor que el azar.

Desventajas: Depende del Umbral de decisión seleccionado para clasificar las predicciones en positivas y negativas.

Área Bajo la Curva (AUC)

Es el default que da Maxent, muchos revisores lo aborrecen por evaluar generalmente los modelos como buenos. El Área Bajo la Curva (AUC) se refiere al área bajo la curva ROC (Receiver Operating Characteristic). La curva ROC muestra la relación entre la tasa de verdaderos positivos (sensibilidad) y la tasa de falsos positivos (1 - especificidad).

Valores: Un AUC = 1 es una predicción perfecta. AUC > 0.5 indica una predicción buena, AUC < 0.5 es una predicción no mejor que el azar.

Ventajas: AUC proporciona una medida del rendimiento del modelo independiente del umbral de decisión.

Desventajas: AUC puede ser menos informativa en casos de clases muy desbalanceadas. Nadie la quiere desde la crítica de Lobo (2008.) Aunque una revisión reciente la defiende (Konowalik & Nosol, 2021).

Área Bajo la Curva Parcial (pAUC)

Variante del AUC que se calcula solo para una parte específica de la curva ROC propuesta por Peterson et al. (2008). Evalua el rendimiento del modelo en regiones específicas de la curva ROC dónde de la tasa de falsos positivos o la tasa de verdaderos positivos son más relevantes.

Valores: de 0 hasta aproximadamente 2. Un pAUC < 1 es una predicción peor que el azar. un valor de pAUC > 1 indica una predicción mejor que el azar. Entre más alto el valor, mejor el modelo.

Ventajas: Facilita la comparación entre modelos en la región de interés, lo que puede ser crucial en aplicaciones como la detección de fraudes, diagnóstico médico, y otras áreas donde los costos de errores varían según el umbral.

Desventajas: Requiere definir la región de interés. Es muy tardada de calcular.

Resumen Comparativo

Métrica Ventajas Desventajas

SDMTune

SDMtune es un paquete de R diseñado para facilitar la creación, evaluación y optimización de modelos de distribución de especies. Creado por Sergio Vignali, de la universidad de Bern.

Varios manuales y explicación a las funciones están en el Sitio oficial. También pueden ver la publicación.

Características Principales

  • Funciones Clave:

    • Preprocesamiento de datos: separación de datos, obtención del background, visualización.
    • Entrenamiento de modelos: directamente con maxent, especificación de features, regularización, etc.
    • Selección de variables: basada en datos.
    • Evaluación de modelos: varios índices.
    • Optimización de modelos: Experimentos.

Aquí usaremos Maximum training sensitivity plus specificity para transformar a binario. Esto se hace con una reclasificación de los valores del mapa pred

ths_val <- umbrales %>%
  filter(Threshold =='Maximum training sensitivity plus specificity') %>%
  pull(2) # jalamos el valor de la columna 2

m <- matrix(c(0,ths_val, 0, ths_val, 1, 1), nrow = 2, byrow = T)

pred_binario <- classify(pred, rcl = m) 

plot(pred_binario)
