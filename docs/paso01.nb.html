Introducción.

En este documento se describen los pasos para preparar un modelado de distribución de especies en R y se proporcionan referencias para aprender más sobre cada uno de las herramientas y los pasos.

El objetivo de este documento es:

  • Dar un ejemplo reproducible que puedan modificar a su interés.

  • Producir los archivos de registros que se usarán para modelar una especie:

    • training.csv

    • testing.csv

  • Mapa descriptivo de lo anterior

Paquetes requeridos

Manejo de datos

El paquetetidyverse es una colección de paquetes de R diseñados para la ciencia de datos. Proporciona una gramática coherente y herramientas para manipulación y análisis de datos.

Es requerimiento escencial para poder reproducir este script, ya que se usa y abusa de el uso de las pipes (%>%), las cuales enfatizan una secuencia de acciones.

Descarga de datos

Se utiliza la función occ_download, deben tener sus credenciales guardadas. Al hacer esto se especifica claramente el origen de datos y se genera un DOI de la descarga que se citará en cualquier publicación resultante de esto.

gbif_download <- occ_download(
  pred("taxonKey", species_key),
  pred("hasCoordinate", TRUE), # con coordenadas
  pred("hasGeospatialIssue", FALSE), # sin problemas espaciales
  pred_gte("year", 1950),  # a partir de 1950
  pred_in("basisOfRecord",'PRESERVED_SPECIMEN'), # tipo de registro
  # Quitar registros "no nativos"
  pred_or(pred_not(
    pred("establishmentMeans","MANAGED")),
    pred_not(pred_notnull("establishmentMeans"))),
  pred_or(pred_not(pred("establishmentMeans","INTRODUCED")),
          pred_not(pred_notnull("establishmentMeans"))),
  pred_or(pred_not(pred("establishmentMeans","INVASIVE")),
          pred_not(pred_notnull("establishmentMeans"))),
  pred_or(pred_not(pred("establishmentMeans","NATURALISED")),
          pred_not(pred_notnull("establishmentMeans"))),
  format = "SIMPLE_CSV"
  
  # solo agregar si no esta en .Renviron
  # user = gbif_user, 
  # pwd = gbif_pwd,
  #email = gbif_email
)

# Esperar a que esté lista (pueden ser hasta 30 minutos)
occ_download_wait(gbif_download)

Separar en Training (80%) y Testing (20%)

Además, eliminamos todas las columnas con información que no es necesaria para el modelo y conservamos únicamente tres: especie, longitud y latitud.

Training data

set.seed(123) # para reproducibilidad

training <- gbif_cleaner %>%
  select(species, decimalLongitude, decimalLatitude) %>% #seleccionar columnas
  rownames_to_column('ID') %>% # agregar ID para cada punto
  sample_frac(0.80) # porcentaje de training
  
training

Testing data

testing <- gbif_cleaner %>%
  select(species, decimalLongitude, decimalLatitude) %>% #seleccionar columnas
  rownames_to_column('ID') %>% # agregar ID para cada punto
  filter(!ID %in% training$ID)
  
testing
